# fqvl 配置说明

这份仓库提供两套 sing-box 配置：

- `three_ios_1.11.4.txt`：给当前 iOS sing-box `1.11.4` 使用（现在推荐用这个）。
- `three.txt`：给后续更高版本内核使用（客户端升级后再切换）。

## 设计目标

这套配置主要解决 4 件事：

1. 手机上可以手动快速切换模式和节点，不依赖自动测速。
2. 首次启动就能拉规则，避免国内直连 GitHub 失败。
3. 国内外分流清晰，减少误走代理或误直连。
4. 保持结构简单，后续维护改动成本低。

## 核心设计

### 1. 双开关设计（手动可控）

- `mode` 组：`proxy / direct`
  - 相当于总开关，决定默认流量走代理还是直连。
- `proxy` 组：`vless-reality / hy2 / tuic`
  - 相当于节点开关，随时在 App 里手动切。

这样做的目的：你清楚节点质量，就不需要自动测速，直接手动选最稳。

### 2. 规则下载走代理

- `route.rule_set[*].download_detour` 统一设置为 `proxy`。
- 额外加了 GitHub 相关域名强制走代理：
  - `github.com`
  - `githubusercontent.com`

这样做的目的：即使你切到 `direct`，规则更新请求也不会误走直连而失败。

### 3. DNS 与分流策略

- 国内相关规则（`geoip-cn`、`geosite-cn`、`.cn`）走 `direct`。
- 广告域名（`geosite-ads`）直接拒绝。
- 默认策略通过 `mode` 决定，方便你在不同网络环境一键切换。

### 4. 手机端稳定项

- `tun` 使用 `strict_route`、`sniff` 等设置，提高分流命中率和稳定性。
- `cache_file` 打开，减少重复解析/下载带来的抖动。

## 适用场景

适合这些情况：

- 你希望“手动可控”，不要自动测速自动切换。
- 网络环境经常变化（Wi-Fi、蜂窝切换），但希望操作简单。
- 你在国内网络下使用，规则更新需要稳定通过代理。

不太适合这些情况：

- 希望全自动优选节点、完全不手动切换。
- 需要复杂策略（如按 App 精细分流、按时间策略切换等）。

## 首次使用建议（iOS 1.11.4）

1. 先导入 `three_ios_1.11.4.txt`。
2. 进入节点选择，先把 `mode` 设为 `proxy`。
3. 在 `proxy` 组里选你当前最稳的节点。
4. 启动后观察规则是否成功拉取。
5. 确认正常后，再按需要切 `mode` 到 `direct` 做对比。

## 客户端升级后的切换

当 iOS sing-box 升级到支持新版语法后：

1. 保留现有配置备份。
2. 导入 `three.txt`。
3. 按同样方法测试 `mode` 与 `proxy` 切换。
4. 连续使用稳定后再正式替换。

## 安全与维护注意事项

1. 当前仓库里的服务器地址、UUID、密码是示例值，正式使用前请替换为你的真实参数。
2. 不要把真实密钥直接发群或发截图，避免泄露。
3. 规则源如果后续变更，优先改 `rule_set.url`，保持 `download_detour: proxy` 不变。
4. 每次大改后做一次“冷启动 + 切网 + 切节点”三步验收。

## 快速排障

如果出现“规则没更新”：

1. 先确认 `mode=proxy`。
2. 再确认当前 `proxy` 节点本身可用。
3. 看日志里是否有 GitHub 请求失败。
4. 若仍失败，先换另一个节点重试。
